01
이 시스템이 뭔가요?
중학생도 이해할 수 있게 설명합니다
쉽게 말하면
고객이 네이버에서 예약하면 — 자동으로 태블릿에 목록이 뜨고, 현장에서 체크인하고 사진 찍으면 구글 드라이브에 자동 저장되는 시스템입니다.
1차 자동화 (완성)
네이버 예약 메일 → Make → 구글 시트 + 캘린더 자동 기록
2차 자동화 (완성)
아이패드 체크인 + 촬영 → 드라이브 + 시트 자동 저장
02
전체 흐름
처음부터 끝까지 무슨 일이 일어나는지
1
고객 행동
네이버에서 예약
고객이 네이버 예약 서비스에서 날짜·시간·상품을 선택하고 결제까지 완료합니다. 이 순간 네이버가 자동으로 예약 확인 메일을 보냅니다.
네이버 예약
2
자동화 — Make
메일 수신 → 데이터 파싱
Make.com 이 네이버 메일함을 감시하고 있다가 새 예약 메일이 오면 자동으로 잡아챕니다. 메일 내용에서 예약번호, 날짜, 시간, 상품명, 인원수 등을 꺼내냅니다.

시나리오 이름: Integration Email, Google Sheets
Make.com 자동 실행
3
1차 저장
구글 시트 + 캘린더 기록
Make가 꺼낸 예약 정보를 구글 스프레드시트에 한 줄씩 추가합니다. 동시에 구글 캘린더에도 일정이 등록되어 촬영일 알림을 받을 수 있습니다.

저장되는 시트: DK_Reservation_Master → reservations 탭
구글 시트 구글 캘린더
4
현장 운영 — 아이패드
대시보드에서 예약 목록 확인
촬영 당일, 운영 요원이 아이패드 Safari에서 운영 웹앱을 엽니다. 시간 탭(13:00~17:00)을 누르면 해당 타임의 예약 고객 목록이 자동으로 불러와집니다.

앱 주소: script.google.com 앱스 스크립트 웹앱 URL
아이패드 Safari
5
체크인
고객 카드 탭 → 정보 입력
고객이 도착하면 목록에서 해당 고객 카드를 탭합니다. 1차 예약 정보(예약번호, 상품, 시간)가 자동으로 표시됩니다. 운영 요원이 실명, 이메일, 전화번호를 입력하고 개인정보 동의를 받습니다.
실명 확인 동의 수집
6
촬영
촬영 창 열기 → 네임보드 + 카메라
'촬영 창 열기' 버튼을 누르면 팝업 창이 열립니다. 상단에 고객 이름이 크게 표시된 네임보드가 뜨고, 아래에 카메라 화면이 나옵니다. '카메라 시작' → '촬영하기' 버튼으로 머그샷을 찍습니다.

팝업 주소: dksequence.github.io/operation/camera.html
GitHub Pages 팝업 아이패드 카메라
7
자동 저장
드라이브 + 2차 시트 자동 기록
촬영 완료 즉시 3단계가 자동으로 진행됩니다.
① 사진 파일이 구글 드라이브에 저장 (파일명: 날짜_예약번호_실명.jpg)
② 체크인 정보가 2차 스프레드시트에 한 줄 추가
③ 대시보드에서 해당 고객이 '완료' 상태로 변경
구글 드라이브 2차 시트 기록 완료 처리

06
현장 운영 사용법
촬영 당일 운영 요원이 따라하는 순서
1
아이패드 Safari에서 앱 열기
홈화면에 저장된 'DKsequence 현장운영' 아이콘을 탭합니다. 또는 Safari에서 script.google.com 앱 URL을 직접 입력합니다.
2
시간 탭 선택
상단 탭에서 현재 타임(예: 14:00)을 탭합니다. 해당 타임 예약 고객 목록이 자동으로 불러와집니다. 완료된 고객은 흐리게 표시됩니다.
3
고객 카드 탭 → 체크인 화면
도착한 고객의 카드를 탭합니다. 체크인 화면에서 실명을 확인하고 입력합니다. 이메일, 전화번호를 받아서 입력하고 개인정보 동의를 받습니다. (▼ 눌러서 내용 확인 가능)
4
촬영 창 열기 → 카메라 시작 → 촬영
'촬영 창 열기' 버튼 탭 → 팝업 창에서 '카메라 시작' 탭 → 권한 허용 → '촬영하기' 버튼으로 머그샷 촬영. 촬영 완료 후 팝업이 자동으로 닫힙니다.
5
자동 저장 확인 후 다음 고객
저장 중 화면에서 3단계(드라이브 저장 → 시트 기록 → 완료처리)가 순서대로 완료됩니다. '다음 고객' 버튼을 탭하면 대시보드로 돌아갑니다.
현장 결제 고객 (예약 없는 손님)
대시보드 하단 '+ 신규 고객 추가' 버튼을 탭합니다. 접수 코드가 자동 생성(20260408-W1 형식)되고 이름, 시간, 상품, 결제 방법을 입력 후 촬영하면 됩니다.

08
문제 해결 가이드
뭔가 안 될 때 여기를 보세요
예약 목록이 불러오기 실패로 나와요
1차 스프레드시트(DK_Reservation_Master)에 해당 날짜 데이터가 있는지 확인하세요. 앱스 스크립트에서 배포가 최신 버전인지 확인하세요. (배포 → 배포 관리 → 버전 확인)
카메라가 안 열려요
아이패드 설정 → Safari → 카메라가 '묻기' 또는 '허용'으로 되어 있는지 확인하세요. 팝업 창이 차단됐다면 Safari 설정에서 팝업 허용으로 바꿔주세요.
사진 저장은 됐는데 시트 기록이 안 돼요
앱스 스크립트 실행 로그에서 오류를 확인하세요. (code 편집기 → 실행 로그 탭) 2차 스프레드시트 ID가 Code.gs CONFIG에 올바르게 입력되어 있는지 확인하세요.
Make 자동화가 멈춘 것 같아요
make.com에 접속해서 'Integration Email, Google Sheets' 시나리오가 켜져 있는지 확인하세요. (오른쪽 토글이 파란색이면 ON) 실행 기록에서 오류가 있는지 확인하세요.
코드 수정 후 아이패드에 반영이 안 돼요
앱스 스크립트 코드 수정 후에는 반드시 재배포가 필요합니다. 배포 → 배포 관리 → 연필 아이콘 → 새 버전 → 배포. Safari에서 새로고침하면 반영됩니다.

001 What is this system?
Explained simply enough for a middle school student to understand
Simply put
When a customer books through Naver, the booking automatically appears on the tablet list, and after on-site check-in and photo capture, the files are automatically saved to Google Drive.
Primary Automation (Completed)
Naver booking email → Make → automatic logging to Google Sheets + Calendar
Secondary Automation (Completed)
iPad check-in + capture → automatic save to Drive + Sheet

002 Complete Flow
What happens from start to finish
1
Customer Action
Book on Naver
The customer selects a date, time, and product in the Naver booking service and completes payment. At that moment, Naver automatically sends a booking confirmation email.
Naver Booking
2
Automation — Make
Receive email → parse data
Make.com monitors the Naver mailbox and automatically catches new booking emails. It extracts the booking number, date, time, product name, number of people, and other details from the email.

Scenario name: Integration Email, Google Sheets
Make.com Runs automatically
3
Primary Storage
Log to Google Sheets + Calendar
The booking data extracted by Make is added to Google Sheets one row at a time. At the same time, an event is added to Google Calendar so you can receive reminders for the shoot date. Stored in sheet: your_sheet → reservations tab
Google Sheets Google Calendar
4
On-Site Operations — iPad
Check booking list in dashboard
On the day of the shoot, the operator opens the operations web app in Safari on the iPad. Tapping a time tab (13:00–17:00) automatically loads the list of customers booked for that slot.

App URL: script.google.com Apps Script web app URL
iPad Safari
5
Check-in
Tap customer card → enter information
When the customer arrives, tap that customer card in the list. The primary booking info (booking number, product, time) appears automatically. The operator enters the real name, email, and phone number, and collects privacy consent.
Verify real name Collect consent
6
Capture
Open capture window → name board + camera
Pressing the “Open Capture Window” button opens a popup. A name board with the customer’s name appears large at the top, and the camera view appears below. Use “Start Camera” → “Take Photo” to capture a mugshot. Popup URL: your_id.github.io/operation/camera.html
GitHub Pages popup iPad camera
7
Auto-save
Drive + Secondary Sheet Auto-logging
As soon as capture is complete, three steps run automatically.
① The photo file is saved to Google Drive (filename: date_bookingnumber_realname.jpg)
② The check-in info is added as one row in the secondary spreadsheet
③ That customer is marked as “Completed” in the dashboard
Google Drive Secondary Sheet Log Completion status

006 How to Operate On Site
Steps for the operator on the shoot day
1
Open the app in iPad Safari
Tap the “DKsequence On-Site Operations” icon saved on the home screen, or enter the script.google.com app URL directly in Safari.
2
Select a time tab
Tap the current time tab at the top (for example, 14:00). The booking list for that time slot loads automatically. Completed customers appear dimmed.
3
Tap customer card → check-in screen
Tap the card of the arriving customer. On the check-in screen, verify and enter the real name. Enter the email and phone number, and collect privacy consent. (Press ▼ to view the details.)
4
Open capture window → start camera → capture
Tap “Open Capture Window” → in the popup tap “Start Camera” → allow permission → use “Take Photo” to capture the mugshot. After capture, the popup closes automatically.
5
Confirm auto-save, then move to the next customer
On the saving screen, the three steps (save to Drive → log to sheet → mark as completed) finish in order. Tap the “Next Customer” button to return to the dashboard.
On-site paying customers (walk-ins without a booking)
Tap the '+ Add New Customer' button at the bottom of the dashboard. A reception code is generated automatically (format: 20260105-W1), then enter the name, time, product, and payment method and proceed with capture.

008 Troubleshooting Guide
Check here when something is not working
The booking list shows a loading failure
Check whether the primary spreadsheet (DK_Reservation_Master) contains data for that date. Also confirm that the Apps Script deployment is the latest version. (Deploy → Manage deployments → check version)
The camera will not open
Go to iPad Settings → Safari → make sure Camera is set to “Ask” or “Allow.” If popups are blocked, change Safari settings to allow popups.
The photo was saved, but the sheet was not updated
Check the error in the Apps Script execution log. (Code editor → Execution log tab) Also verify that the secondary spreadsheet ID is correctly entered in the Code.gs CONFIG.
It seems the Make automation has stopped
Go to make.com and check whether the “Integration Email, Google Sheets” scenario is turned on. (If the toggle on the right is blue, it is ON.) Also check the run history for errors.
Changes are not reflected on the iPad after editing the code
After editing Apps Script code, you must redeploy. Deploy → Manage deployments → pencil icon → New version → Deploy. Then refresh Safari to see the changes.

009 How to Redeploy After Editing Code
When you edit Code.gs or index.html

Open script.google.com
Open the 20260101_YOUR_DOC_v101 project.
2
Edit the code and save
Edit Code.gs or index.html and save with Ctrl+S.
3
Deploy → Manage deployments
Click the “Deploy” button in the upper-right corner → click “Manage deployments” (not New deployment!)
4
Pencil icon → New version → Deploy
Click the pencil (✏️) to the right of the existing deployment → select “New version” from the version dropdown → click the “Deploy” button
5
When editing camera.html — GitHub only
camera.html exists only on GitHub. github.com/YOUR_ID/YOUR_ID.github.io → operation/camera.html → ✏️ Edit → modify → Commit changes
Important
Never change the three values in the CONFIG block of Code.gs: ROOT_FOLDER_ID, SHEET_1ST_ID, and SHEET_2ND_ID. If they change, the data will be saved in the wrong place.

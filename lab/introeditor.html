001 What is this tool?
A point-and-click editor for HTML files that runs entirely inside your browser
The simplest possible explanation
You have an HTML file — a webpage, a document, a flyer. You want to change the text, tweak a color, or rearrange a section. Normally that means opening a code editor and hunting through hundreds of lines of markup. This tool lets you skip all of that.
Open the editor in your browser, drag your HTML file in, and click directly on the element you want to change. The right panel immediately shows you everything editable about that element — text, colors, font size, spacing — and any change you make reflects on the page in real time. When you're done, click Save and download the updated HTML file.

003 How to use — step by step
From zero to edited file in under two minutes
1
Open the editor
Double-click html-editor-EN.html to open it in Chrome or Safari. No installation, no account, no internet connection required.
2
Load your HTML file
Click Open File in the toolbar, or simply drag and drop your .html file onto the drop zone. The page renders immediately in the canvas.
3
Click what you want to edit
Hover over any part of the page — a blue dashed outline appears around elements. Click the one you want. It becomes highlighted in blue, and the right panel fills with all its editable properties.
4
Make your changes
Type new text in the Content field. Pick a color from the swatches. Adjust font size, bold, alignment, margins. Every change updates the canvas in real time — what you see is what you get.
5
Preview in a real browser tab
Click Preview anytime to open the current state of your document in a new tab — exactly as it would look when published. No need to save first.
6
Save your file
Click Save HTML when done. The browser downloads yourfilename_edited.html — a clean, complete HTML file with your original styles and structure fully preserved, ready to upload or share.
Keyboard shortcuts
Ctrl + Z (or Cmd + Z on Mac) — Undo the last change, up to 40 steps back.   Ctrl + S — Save the file instantly without reaching for the button.
04 Feature overview
Everything you can do once a file is loaded
✏️
Text editing
Edit the text content of any element — headings, paragraphs, links, buttons. Multi-line text supported. Changes reflect instantly.
🎨
Color picker
One-click color swatches for text color and background color. Eight preset options for each. Works on any element.
🔤
Font controls
Bold, italic, underline toggles. Font size in one click from 12px to 48px. Left / center / right / justify alignment.
📐
Layout controls
Set margin, padding, width, and height directly — type any valid CSS value. Perfect for spacing adjustments without touching code.
🖼️
Image & link editing
Change image source (src) and alt text for any image. Update href link targets. Works with both local and remote URLs.
⌨️
Raw CSS input
For those rare cases where you need more control — type any CSS directly into the style field and it applies immediately.
➕
Add Block
Insert new elements — paragraph, heading, image, list, link, divider, custom HTML — before, after, or inside any selected element.
⬜
Spacer insert
One-click to add a 32px blank spacing div after the selected element. Labeled in the editor, outputs as clean empty div on save.
↕️
Move elements
Reorder any element up or down within its parent using the ▲ and ▼ toolbar buttons. The structure tree updates instantly.
🗑️
Delete elements
Remove the selected element with a confirmation prompt. Combine with Undo if you delete something by mistake.
↩️
Undo (40 steps)
Every edit is recorded. Press Ctrl + Z or click Undo to step back through your changes, up to 40 actions.
👁️
Preview tab
Open the current document in a new browser tab at any time to see exactly how it will look when published — with all editor markers removed.
What it doesn't do
This tool edits the content and inline styles of existing elements. It doesn't modify the document's <style> stylesheet or restructure deeply nested layouts. For those needs, a code editor like VS Code is the right tool.
05 Real scenarios
Three situations where this tool saves significant time
Scenario A — Photographer
Updating a portfolio page before an exhibition
Mia is a photographer preparing for a group show. Her portfolio is a single HTML file she received from a designer two years ago. The show details have changed — new venue, updated dates, one image replaced. She's never opened the code.
"I just opened the editor, clicked on the venue name, typed the new one, clicked on the date — done. Found the old image in the tree, swapped the src. Total time maybe eight minutes. Then hit Preview to double-check and saved."
Previously this would have meant emailing the designer, waiting a day, and paying a revision fee. With the editor, she did it herself on the morning of the opening, on her phone using Safari.
Scenario B — Studio Operations
Weekly update of a booking information document
A photography studio sends clients a branded HTML document before each shoot — pricing, location, what to bring. The content is 80% the same every week but a few details always change: the specific date, a product name, a contact note.
"Before, whoever handled admin had to ask me to update the file every week. Now they open the editor themselves, change the three fields that need changing, hit save, and send it. The structure and design stay exactly the same."
The studio now has a clean workflow: one person owns the design, another updates the content — no handoffs, no risk of accidentally breaking the layout.
Scenario C — Event Organizer
Last-minute corrections to an event flyer
An arts collective built a beautiful HTML flyer for a pop-up event. Two hours before the event, the venue entrance changed and a performer cancelled. Updating the HTML flyer was urgent — but the person who built it wasn't reachable.
"Someone on the team had never touched HTML in their life. We opened the editor, I pointed at the address on screen and said 'click that'. She clicked it, it highlighted, she typed the new address. Done in 30 seconds."
The flyer was updated, re-saved, and the new link sent out — all within five minutes, by someone with zero prior HTML experience.

07 FAQ
Common questions
I clicked on the page but nothing gets selected
Make sure you have a file loaded first — the drop zone should be hidden and the canvas should show your HTML content. If you opened the editor file itself (not your HTML file), load your file via Open File or drag-and-drop. Once a file is loaded, clicking anywhere in the canvas selects elements.
I edited the text but the canvas didn't change
The Content field uses the oninput event — it updates as you type, not on Enter. Make sure you're typing into the Content textarea in the right panel (it should have a blue focus border). If the element has complex child markup, text changes will replace its inner content with plain text.
The saved file looks different from what I saw in the editor
The editor canvas inherits some base styles from the editor itself. Use the Preview button (not the canvas) to see the true saved output before downloading. The Preview opens a fully isolated version of your file — exactly as it will look when opened independently.
I accidentally deleted something. Can I get it back?
Yes — press Ctrl + Z (or Cmd + Z on Mac) immediately. The editor stores up to 40 undo steps. Note that undo history is lost if you reload the page, so save frequently when making major structural changes.
Can I use this on my phone or tablet?
Yes, it works in Safari on iPad and iPhone. The layout adapts for smaller screens — the right properties panel is hidden on very small displays but can still be accessed by selecting elements. For comfortable use, a tablet or larger screen is recommended.
Will my HTML file's original CSS and fonts be preserved?
Yes. The editor extracts and stores your file's entire <head> section — including all <style> blocks, font imports, and meta tags — and re-inserts it untouched when you save. Only the body content is editable; the stylesheet is never modified.
